SummaryWhat we’re changing and why

The page is already accuracy-led with strong benchmark tables (WebWalker 81% vs 71%, MKQA 70% vs 63%) and the right key-differentiator H3s. Three things are missing:

  • Post-Nebius positioning. Tavily was acquired by Nebius — buyers researching alternatives need to see what that means for their roadmap exposure. A new section above the Feature Comparison addresses it head-on.
  • Migration path. The page currently ends in a generic “Try API for free” CTA. A “Migrating from Tavily” section with a code diff is the single most useful conversion element for the active-evaluation buyer.
  • FAQPage schema. The page already has a FAQ section but no JSON-LD. Adding schema is a one-edit win that unlocks AI overview citation eligibility.

NewH2Migrating from Tavily — drop-in guide

New section, code-led. Show the Tavily call signature and the Exa call signature side-by-side so a dev can see in 15 seconds that the migration is a one-line change.

# Tavily
from tavily import TavilyClient
client = TavilyClient(api_key="...")
result = client.search("openai gpt-5 launch date")

# Exa — same shape, neural by default
from exa_py import Exa
client = Exa(api_key="...")
result = client.search("openai gpt-5 launch date")

This is the single most useful section for an active-evaluation buyer. Owns the “tavily alternative” and “migrate from tavily” long-tail.
