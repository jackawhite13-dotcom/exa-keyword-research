SummaryWhat we’re changing and why

The integration pages today are partner-listing templates — hero, overview, quick links, related partners. They convert poorly on organic because they don’t answer the actual query a dev landing on the page has, which is “how do I add web search to my OpenAI agent in 30 seconds?” The deepening rewrite makes that the page.

  • Title and H1 lead with the head term, not the brand pair. “OpenAI Web Search Tool with Exa” vs “Exa + OpenAI.”
  • Code snippet front-and-center. Above-the-fold. 3-line setup. This is the single highest-value change.
  • Add a how-to walkthrough with HowTo schema. “Step 1 / Step 2 / Step 3” structure earns rich snippet eligibility and Gemini / ChatGPT citation.
  • Six-question FAQ with FAQPage schema. Same playbook as the /exa-api hub.

NewCodeAbove-the-fold setup snippet

The single most important change on this page. Code block before any prose — dev visiting cold sees the snippet first.

from openai import OpenAI
from exa_py import Exa

openai = OpenAI()
exa = Exa(api_key="...")

# Register Exa as an OpenAI tool
tools = [exa.openai_tool()]

response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What launched today in AI?"}],
    tools=tools,
)

Three things the snippet should communicate: it’s short, the SDK does the OpenAI function-spec wiring for you, and the result is a real call against GPT-4o (not a stub).

NewH2How to add OpenAI web search with Exa (step-by-step)

HowTo-structured walkthrough. Three steps. Eligible for HowTo rich result.

  • Step 1: Install the SDKs (pip install openai exa_py)
  • Step 2: Register the Exa tool with your OpenAI client (exa.openai_tool())
  • Step 3: Call chat.completions.create with tools=tools and let GPT-4o trigger the search when it needs the web

Each step gets a code block. Optionally a screenshot of the trace in the OpenAI dashboard.
