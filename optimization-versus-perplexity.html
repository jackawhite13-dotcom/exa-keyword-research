01

What we are changing and why

Lightest touch of the four existing page mockups. The page already mentions both Search API ($5/1K) and Sonar. The H3 on flat pricing is well targeted. Three deepening moves.

  • Promote Sonar Pro to its own H3. Captures “sonar pro” (150 vol, KD 5) and “perplexity sonar api” (400 vol).
  • Add a developer vs consumer positioning paragraph above the table. The one line framing buyers carry away.
  • FAQPage schema on the existing FAQ. Single edit, unlocks AI overview citation.
Step 02

Title tag and meta description

Current title
Exa vs Perplexity API: Neural Search vs Black Box Answers (2026)
Proposed title
Exa vs Perplexity API (Sonar), Neural Search Comparison (2026)
Why: “Sonar” in the title captures the active brand search now that Perplexity is pushing Sonar as the API name. “Black Box Answers” reads editorial. Drop it. “Neural Search Comparison” aligns with /versus/tavily.

Not explicitly set.

Comparing Perplexity Sonar and Exa for AI agents? Exa scores 64.8% vs 60.1% on a 500 query eval, returns in 1.5s vs 5.1s, and ships full page content vs 4,096 token snippets.

NewH3

Sonar Pro, what it is and when to use it

New H3 under Key Differentiators. Captures “sonar pro” (150 vol, KD 5) and “perplexity sonar api” (400 vol) explicitly.

Draft copy. “Sonar is Perplexity's LLM synthesized answer endpoint. Sonar Pro adds longer outputs and stronger reasoning at a higher price tier. Both are good for chat style answers but trade transparency for synthesis. You get a generated response instead of the retrieval results you can re rank, filter, or feed into your own LLM. Exa's API returns the underlying retrieval so your agent owns the synthesis step.”
