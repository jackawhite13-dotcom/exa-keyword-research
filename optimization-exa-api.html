01

What we are changing and why

The page has strong product content. Benchmarks, customer logos, security signals. It is structured around a brand narrative (“The web, organized for agents”) rather than the head term query. Three moves do the bulk of the work.

  • Title and H1 lead with the head term. The proposed H1 anchors the keyword first and keeps the brand line as a subheading.
  • Add a definition block right under the hero. “What is a web search API?” captures the featured snippet and feeds the AI overview.
  • Add a FAQ section with FAQPage schema. Six questions covering meta, competitor comparisons, and latency. The single biggest GEO lever on the page.
Step 02

Title tag and meta description

Lead with the head term, keep the brand suffix, stay under 60 characters.

Current title
Exa | Web Search API, AI Search Engine, & Website Crawler
Proposed title
Web Search API for AI Agents and LLMs, Exa
Why: Brand first titles compete with Google's rewriter and lose head term weight. Leading with the keyword plus an ICP qualifier (“for AI Agents and LLMs”) keeps the primary first, signals the segment, and trails the brand as the suffix. A high ranking pattern in this peer set (SerpAPI, Tavily).
Current meta description
Not explicitly set. Google generates one from page copy.
Proposed meta description
Exa's web search API gives AI agents real time results from a billion page neural index. p95 latency 1.7s, 1,200 domain filters, SOC 2 Type II. Try free.
Why: Sets click through expectations on three benchmark numbers Exa already publishes. The “Try free” CTA in the meta lifts CTR in commercial SERPs.

NewH2

What is a web search API? (definition block)

New section directly under the hero. A 60 to 90 word answer to the head term's implicit definitional question. Optimized for the featured snippet and the AI overview citation.

Draft copy. “A web search API is a programmatic interface that returns ranked results from a live index of the public web. Unlike scraping a search engine, a web search API exposes its own retrieval layer (filters, freshness controls, structured outputs) so AI agents and applications can pull data without rendering a browser or rotating proxies. Exa's web search API runs on a neural index of over a billion pages, returns p95 in under 1.7 seconds, and supports 1,200 domain filters.”
