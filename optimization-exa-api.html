Page optimization mockup

/exa-api

Theme 1 — the head-term category hub. Scott named “web search api” as a money keyword on Call 1. The page targets it in the title today but doesn’t rank top. This mockup is the deepening pass to convert that targeting into a top-ranking page.
Theme: 1 of 12
Goal: Goal 1 · Product queries
Primary keyword: web search api
Status: Existing · deepen

SummaryWhat we’re changing and why

The page has strong product content (benchmarks, customer logos, security signals) but it’s structured around a brand narrative (“The web, organized for agents”) rather than the head-term query. Three changes do the bulk of the work:

  • Title + H1 lead with the head term. Today the H1 is “The web, organized for agents.” The proposed H1 anchors the keyword first and keeps the brand line as a subheading.
  • Add a definition block right under the hero. “What is a web search API?” captures the featured snippet for the head term and feeds the AI overview.
  • Add a FAQ section with FAQPage schema. Six questions covering the head-term meta + competitor comparisons + latency/freshness. This is the single biggest GEO lever.
Keep Modify New Schema

01Title tag & meta description

The title tag is the single biggest on-page ranking signal. Lead with the head term, keep the brand suffix, stay under 60 characters.

Current title
Exa | Web Search API, AI Search Engine, & Website Crawler
Proposed title
Web Search API for AI Agents & LLMs — Exa
Why: Brand-first titles (“Exa | …”) compete with Google’s rewriter and lose head-term weight. Leading with the keyword + ICP qualifier (“for AI Agents & LLMs”) keeps the primary first, signals the segment, and trails the brand as the suffix — a common high-ranking pattern in this peer set (SerpAPI, Tavily).
Current meta description
Not explicitly set — Google is generating one from page copy.
Proposed meta description
Exa’s web search API gives AI agents real-time results from a billion-page neural index. p95 latency 1.7s, 1,200-domain filters, SOC 2 Type II. Try free.
Why: Sets click-through expectations on three benchmark numbers Exa already publishes. The “Try free” CTA on the meta lifts CTR in commercial SERPs.

NewH2What is a web search API? (definition block)

New section directly under the hero. 60-90 word answer to the head-term’s implicit definitional question. Optimized for the featured snippet + AI overview citation.

Draft copy: “A web search API is a programmatic interface that returns ranked results from a live index of the public web. Unlike scraping a search engine, a web search API exposes its own retrieval layer — including filters, freshness controls, and structured outputs — so AI agents and applications can pull the data they need without rendering a browser or rotating proxies. Exa’s web search API runs on a neural index of over a billion pages, returns p95 in under 1.7 seconds, and supports 1,200-domain filters.”
